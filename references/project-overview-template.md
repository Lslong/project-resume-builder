# 项目全景介绍模板(project-overview-template)

附加产物按本文件产出 `项目介绍.md`。

## 定位(先划清边界，避免和另两份产物重叠)

这份文档是**中性客观的项目全景文档**，类似项目初始化生成的 CLAUDE.md。目的只有一个：让读者(候选人自己 / 也可给协作者)对项目建立**整体认知**——是什么、怎么搭的、数据怎么流。

和另两份产物的分工：

| **产物** | **回答的问题** | **语气** |
| --- | --- | --- |
| 简历素材.md | 我做的哪几点最值得写进简历 | 求职卖点，精炼 |
| 面试稿.md | 面试官会怎么追，我怎么接 | 备战弹药，求全 |
| 项目介绍.md(本文件) | 这个项目整体长什么样 | 项目文档，中性 |

**硬边界：** 本文件**不写求职卖点、不写"面试官可能追问"、不编造数字**。亮点只做速查索引并指向面试稿，不重复展开追问逻辑。一旦开始写"为什么不用方案 B、面试官会怎么问"，就是越界到面试稿的活了。

## 产物结构(按此顺序)

### 一、项目速览

```plain
【项目名】· 一句话定位(是什么 + 解决什么问题)
技术栈:技术A / 技术B / 技术C / ...
```

一段话(3-5 句)概述：项目背景、解决的核心问题、整体形态(Web 应用 / 中台 / SDK / 小程序…)。让读者 10 秒内知道这是个什么东西。

### 二、技术栈详单

分层列清，而非堆一行关键词。来源：依赖文件 + 配置 + import，均需代码实证，**不臆测未使用的技术**。

```plain
- 语言 / 框架:……
- 状态管理:……
- 构建 / 工程化:……
- 后端 / 接口:……(若有)
- 存储 / 数据:……(若有)
- 关键三方库:……(只列对架构有影响的，不罗列全部依赖)
```

### 三、整体技术架构

讲清分层方式、模块边界、各目录/模块职责。**用文字版架构图**(ASCII / 缩进树)，不贴源码：

```plain
┌──────────────┐
│   表现层      │  页面 / 组件:……
├──────────────┤
│   逻辑层      │  状态管理 / 业务 hook:……
├──────────────┤
│   数据层      │  请求封装 / 缓存:……
└──────────────┘
```

配一段文字说明各层职责与边界依据。再用一个简表列主要目录/模块各做什么：

```plain
- src/xxx/ —— 职责……
- src/yyy/ —— 职责……
```

### 四、核心流程

挑 1-3 条**最能代表项目骨架的主流程**(关键业务流 / 核心技术流)，步骤化描述。用编号步骤或文字时序，不贴源码：

```plain
核心流程:XXX
1. 用户触发 …… →
2. 经过 …… 处理 →
3. …… →
4. 最终 ……
```

每步说清"做什么、由哪个模块负责"，让读者能在脑中跑通一遍。

### 五、数据流

数据**从哪来 → 怎么流转 → 到哪去**。覆盖请求链路、状态流转、存储落点中项目实际存在的部分：

```plain
数据源(接口/本地) → 请求层封装 → 状态管理 → 组件消费 → 用户交互回写 → ……
```

用箭头图 + 文字说明关键节点的转换/缓存/同步逻辑。这是最能体现"理解了项目"的部分，务必基于代码实证，不脑补。

### 六、技术亮点速查

只做**索引**，不展开：列出 1-N 个亮点的一句话标题，标注"详见 面试稿.md 对应条目"。目的是让读者知道项目里哪些点有深度，具体怎么讲去看面试稿，避免和面试稿重复。

```plain
- 亮点:XXX(详见面试稿)
- 亮点:YYY(详见面试稿)
```

### 七、(可选)关键模块伪代码示意

仅当某个核心机制**用自然语言确实讲不清结构**时，才放一段伪代码辅助，并在开头标注"以下为伪代码，非项目真实源码"。能用文字 + 图讲清的，一律不要上代码。

## 写作纪律

+ **零真实源码**：不贴项目任何真实代码。需要代码示意处一律用伪代码 + 自然语言，并显式标注"伪代码"。优先用文字图(架构图 / 数据流图)替代代码。
+ **基于实证，不脑补**：架构、流程、数据流必须能从代码看出来。看不出的不写，不为了完整而臆造环节。
+ **中性客观**：像项目文档，不是简历。不写卖点、不写追问、不编数字。涉及数字(规模、性能)若代码里无从得知，直接不写，不留占位符——占位符是简历的机制，介绍文档不需要。
+ **守边界**：亮点只索引、指向面试稿；一旦写起"为什么这么选 / 面试官会追什么"，停下，那是另两份产物的职责。
+ **篇幅适度**：全景清晰即可，不堆砌全部细节。读者读完能在脑中重建项目骨架，就达标了。

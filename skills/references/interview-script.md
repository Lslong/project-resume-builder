# 面试备战弹药库模板(interview-script)

阶段 3 按本文件产出 `面试稿.md`。

定位：这是给候选人自己临场备战用的弹药库，不是给面试官看、也不是逐字背诵稿。所以覆盖求全、宁多勿缺，候选人临场自主挑用哪些。与简历相反：简历给面试官看 → 精炼；面试稿给自己 → 求全。

## 一、项目讲解提纲(非背诵稿)

只列结构，让候选人理解怎么讲，而非背台词：

+ 背景：1 句(项目是什么、解决什么问题)
+ 你负责什么：1-2 句
+ 技术亮点：1-2 个
+ 难点：1 个

*提示候选人：面试官会随时打断追问，讲解是骨架不是台词，理解结构即可。*

## 二、技术点追问弹药库(覆盖全，按项目复杂度伸缩)

覆盖项目里所有有价值且符合标准的技术点(不限于简历精选的 3-5 条)。项目越复杂、技术点越多，这里写越多。

每个技术点一个条目，统一结构：

```plain
### 技术点:XXX
- 一句话内核:(候选人必须能脱口复述的)
- 最可能被追的问题(由浅入深,一路追到答不出为止):
  1. ……
  2. ……
  3. ……
- 答题骨架(要点,不是背稿):
  - ……
- 风险提示:(若有讲不稳/容易被问倒的地方,标出来)
```

不分"一面 / 二面 / 压力"层——真实追问不分层，是顺着一个点一路深挖。按"由浅入深"排列问题即可，模拟面试官顺着追的过程。

## 三、亮点设计专区

收录能体现技术实力、但套不进"方案推演"框架的精巧设计(优雅抽象、巧妙数据结构、关键性能手法、清晰解决复杂问题的思路等)。不要因为套不进推演框架就漏掉这些亮点。

每个亮点一个条目：

```plain
### 亮点:XXX
- 巧在哪:
- 解决了什么问题:
- 怎么主动讲给面试官:(如何在合适时机自然抛出这个亮点)
- (可选)伪代码辅助:仅当该点可能被要求白板手写时,用伪代码 + 自然语言,并标注"以下为伪代码"
```

## 四、写作纪律

+ 不背稿：全是骨架 / 要点，不是逐字台词。
+ 不编造：数字用占位、不编亲历经历，与简历素材同一套诚信红线(见 `scan-rubric.md` 第六节)。
+ 伪代码边界：仅"可能被要求白板手写"的技术点可用伪代码，且明确标注；其余一律自然语言描述。
+ 覆盖优先：有价值的技术点尽量都收，采纳权交候选人。无技术含量的不收。
